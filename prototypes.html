Prototypes

Paper Prototypes

Paper prototypes were created to test designs, favoured as a quick method of emulating interfaces. This model also encourages users to focus less on aesthetics and more so its functions and flow of the application.

A pre-testing survey was conducted to identify user backgrounds and to check factors that affect user testing results, such as their previous experience with ambulances and their proficiency in using health related applications.

Post-testing surveys were given to users as two separate surveys; individual forms for each mobile and smartwatch interface. User evaluation of interfaces were recorded using a rating scale as well as short-answer questions.

The main user evaluation method used was the think aloud protocol, allowing designers to observe users whilst interacting with the interfaces, giving designers live direct feedback. Users were asked to complete tasks as they were observed whilst being timed, errors and comments being recorded with design heuristics being evaluated through interactions with the interface.

First Iteration

As the first iteration, two potential interfaces for users to interact with were created. The purpose of this was to evaluate which received more positive feedback from users as being a more user-friendly interface.

The mobile application involved the user manually triaging their injury to be able to identify the severity of the injury and decide regarding if they wished to call an ambulance or not. By allowing the device to autonomously make the decision for the user regarding if they required an ambulance, it automatically removes the need for unnecessary calls. Personal information was locked behind biometrics and a pin with many tasks revolving around being able to distinguish the status of being logged into the correct account.

When user testing there was a great demand for a home screen to enable user freedom control, allowing users to freely 'exit' when needed. The main camera screen when presented was met with confusion as certain users were unaware how to operate the interface. Design elements were also questioned as there was a mismatch between the system and real world designs, especially in terms of icons

The smartwatch application involved triaging passively using the body's vital signs, requiring no active input from the user. Many users when testing the paper prototype found that the amount of text and information was too overwhelming for the smaller screen estate size of a smartwatch. Navigating a smartwatch was also something majority of testing participants were not experienced with, resulting in numerous navigational issues as they were not sure if to swipe or press a button.

Survey results showed that users found the smartwatch easier to navigate than the phone application; even when considering this paper prototype being their first interaction with a 'smartwatch'.

Second Iteration

The second iteration was created with heavy consideration using feedback from the user evaluation sessions. Designs for both interfaces were adjusted, adding in suggested features such as a home screen to address voiced heuristic concerns. Major improvements were made to the navigational aspect of the application, making it easier to access pages and return to major screens. Other prevalent issues such as icon designs and documentation were also fixed, creating a better match between system and the real world.

All results were compiled as a team to identify common issues users faced. Despite changing the icons in this iteration, not all users could understand what the icon was supposed to represent. There were also requests for better help and documentation on both devices, with users expressing their frustration with the lack of instructions provided. Users evaluated the smartwatch as being easier to understand once again, however there was an improvement for mobile navigation.

Third Iteration

The smartwatch interface was chosen as the main device as majority of users preferred the smartwatch over the mobile application in the post experience survey; even when considering lack of experience from users. Some users stated that they felt as though smartwatches were more 'believable' due to many health applications being created for wearables. Other users stated that if they were in severe pain that it was highly unlikely to be capable of operating a device to triage themselves.

A paired equivalent to the smartwatch application was created as majority of smartwatch applications don't exist as a standalone. With the successful login process that the second iteration of the mobile application presented, it was decided that the final solution would log in using a mobile application. Screens were created on photoshop as image files.
